Lesson 1.3: Relations vs. Functions - Summary

Key Concepts: Relations vs. Functions

Overview

This lesson covers the fundamental measures of center used to analyze and summarize data sets. You'll learn how to calculate and apply the mean, median, and mode, and understand when each is most appropriate.

Key Definitions

What is a 'relation' in mathematics?

A relation is any set of ordered pairs (x, y). It may or may not be a function.

What is the defining property of a 'function'?

A function is a relation in which each element of the domain (input) is paired with exactly one element of the range (output).

What is the 'domain' of a function?

The domain is the complete set of all possible input values (x-values) for which the function is defined.

What is the 'range' of a function?

The range is the set of all actual output values (y-values) that the function produces.

What is a 'mapping diagram'?

A mapping diagram displays domain elements on one side and range elements on the other, with arrows showing which inputs map to which outputs.

What is the difference between a 'function' and a 'one-to-one function'?

Every one-to-one function is a function, but a one-to-one function adds the requirement that no two different inputs produce the same output.

Real-World Applications

Example: A parking garage charges a flat rate based on the number of hours parked: $5 for 0-2 hours, $10 for 2-4 hours, $15 for 4-6 hours. Is the cost a function of hours parked?

Each input (hours parked) maps to exactly one cost, making it a function. The fact that multiple hour values share the same cost does not violate the function definition.
