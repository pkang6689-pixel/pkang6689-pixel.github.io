Lesson 9.6: Applications in Image Processing - Summary

Key Concepts: Applications in Image Processing

Overview

This lesson covers the fundamental measures of center used to analyze and summarize data sets. You'll learn how to calculate and apply the mean, median, and mode, and understand when each is most appropriate.

Key Definitions

What is an 'image kernel' (filter)?

An image kernel is a small matrix (typically 3x3 or 5x5) whose entries are weights. Different kernels achieve different effects: blurring, sharpening, edge detection, embossing, etc.

What is 'image sharpening' in terms of matrix operations?

Unsharp masking adds back the high-frequency content: A_sharp = (1+alpha)*A - alpha*A_blur. The Laplacian kernel [[0,-1,0],[-1,4,-1],[0,-1,0]] directly detects and enhances edges through convolution.

What is 'image segmentation' using spectral methods?

Spectral segmentation builds a graph where pixels are nodes and edge weights represent similarity. The eigenvectors of the graph Laplacian (especially the Fiedler vector) reveal natural partitions of the image.

What is 'image registration' from a linear algebra perspective?

Image registration finds a transformation matrix T that maps points in one image to corresponding points in another. The 2D affine transformation is a 2x3 matrix, and finding it involves solving a least-squares system.

What is 'homography' in computer vision?

A homography H is a 3x3 matrix (up to scale) that maps points between two views of a planar surface using homogeneous coordinates: x' ~ Hx. It encompasses rotations, translations, scaling, and perspective distortions.

What is 'total variation' denoising?

Total variation (TV) denoising minimizes ||grad(u)||_1 + lambda||u - f||^2 where f is the noisy image. The L1 gradient penalty preserves edges while removing noise, making it superior to simple smoothing.

What is 'morphological image processing' and how does it relate to matrix operations?

Morphological operations use min (erosion) and max (dilation) instead of weighted sums. While not strictly linear, they can be expressed using matrix-like operations with a structuring element acting like a kernel.

A graphics designer applies a color transformation matrix to convert an image from one color space to another. The transformation from RGB to YCbCr uses a 3x3 matrix. If the original pixel is (R, G, B) = (255, 0, 0) (pure red), the Y (luminance) component is approximately 0.299R + 0.587G + 0.114B. What is Y?

Y = 0.299(255) + 0.587(0) + 0.114(0) = 76.245 ~ 76.2. The luminance weights reflect human visual sensitivity (most sensitive to green, least to blue).

Real-World Applications

Example: A grayscale image of size m x n is naturally represented as:

A grayscale image is an m x n matrix where each entry represents the pixel intensity (typically 0-255 for 8-bit images). This matrix representation enables linear algebra operations for image processing.

Example: Image compression using SVD approximates an m x n image matrix A as A_k = U_k S_k V_k^T. If m = n = 500 and k = 20, how many values need to be stored?

Store U_k (500x20 = 10,000) + S_k (20 values) + V_k (500x20 = 10,000) = 20,020 values instead of 250,000. This gives about 12.5:1 compression.

Example: In image filtering, convolution with a 3x3 kernel can be expressed as multiplication by which type of matrix?

2D image convolution can be expressed as multiplication by a block-Toeplitz matrix with Toeplitz blocks. The kernel values appear shifted along diagonals, reflecting the translation-invariant nature of convolution.

Example: The Discrete Fourier Transform (DFT) of an image converts it from the spatial domain to the:

The 2D DFT decomposes an image into a sum of sinusoidal patterns of different frequencies, orientations, and phases. This frequency-domain representation is useful for filtering, compression, and analysis.

Example: In JPEG compression, images are divided into 8x8 blocks and transformed using the Discrete Cosine Transform (DCT). The DCT matrix C for an 8x8 block is what type of matrix?

The DCT matrix C is orthogonal (C^T C = I), meaning the transform is invertible and norm-preserving. The DCT is applied as CXC^T for each 8x8 block X, transforming to a frequency representation.

Example: A color image (RGB) with resolution m x n is represented as:

An RGB color image is represented as three separate m x n matrices for the Red, Green, and Blue channels. SVD or other matrix operations can be applied to each channel independently or jointly.

Example: In image denoising using SVD, noisy pixel values primarily affect which singular values?

Random noise contributes primarily to the smaller singular values because noise lacks the structured patterns captured by the dominant components. Truncating small singular values effectively removes noise.

Example: The eigenfaces approach to face recognition uses PCA on a set of face images. If there are 100 training images each of size 64x64, the eigenfaces are eigenvectors of which matrix?

Each 64x64 image is vectorized to length 4096. The covariance matrix is 4096x4096, and its eigenvectors (eigenfaces) capture the principal variations. In practice, the smaller 100x100 matrix X^T X is used instead.

Example: What does 'image convolution' mean in linear algebra terms?

Image convolution slides a kernel (small weight matrix) over the image, computing weighted sums at each position. This linear operation can be expressed as multiplication by a large sparse Toeplitz matrix.

The DCT represents an image block as a weighted sum of cosine basis functions of increasing frequency. Low-frequency coefficients capture smooth variations while high-frequency ones capture sharp details and noise.

Image inpainting reconstructs missing regions by exploiting structure in the image. Matrix completion methods assume the image has low-rank structure, using observed pixels to infer the missing ones.

Example: A computer vision researcher uses SIFT features, which represent local image patches as 128-dimensional vectors. To match features between two images, she computes the nearest neighbor using Euclidean distance. For 1000 features in each image, how many distance computations are needed for brute-force matching?

Brute-force matching compares every feature in image 1 against every feature in image 2: 1000 * 1000 = 1,000,000 distance computations. This can be optimized using the matrix product X1 * X2^T for the cross-term in Euclidean distance.

Example: A deep learning researcher uses a convolutional neural network where the first layer has 32 filters of size 3x3 applied to a single-channel 28x28 image. How many parameters (excluding bias) are in this layer?

Each filter is 3x3 = 9 parameters, and there are 32 filters: 32 * 9 = 288 parameters. The number of parameters depends on filter size and count, not the input image size.

Example: An autonomous vehicle system processes stereo camera images to estimate depth. The fundamental matrix F relates corresponding points between left and right images by x'^T F x = 0. What are the properties of the fundamental matrix?

The fundamental matrix F is 3x3 with rank 2 (det(F) = 0). It has 9 entries, minus 1 for scale, minus 1 for the det=0 constraint = 7 degrees of freedom. At least 7 point correspondences are needed to estimate it.
